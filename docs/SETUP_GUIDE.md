## الخطوة 1: إنشاء المستودع

إذا لم يكن لديك مستودع بالفعل، قم بإنشاء مستودع جديد على GitHub. اختر اسمًا مناسبًا للمستودع، ويمكنك جعله عامًا أو خاصًا حسب احتياجاتك. إذا كنت تنشئ موقعًا شخصيًا، استخدم الاسم `username.github.io` حيث `username` هو اسم المستخدم الخاص بك على GitHub.

## الخطوة 2: نسخ ملفات سير العمل

قم بإنشاء المجلد `.github/workflows/` في جذر المستودع إذا لم يكن موجودًا. ثم انسخ ملف سير العمل الذي تريد استخدامه إلى داخل هذا المجلد. للمبتدئين، يُنصح بالبدء بملف `simple-deploy.yml` للتحقق من عمل الإعدادات الأساسية.

## الخطوة 3: تفعيل GitHub Pages

اذهب إلى إعدادات المستودع (Settings)، ثم ابحث عن قسم Pages في الشريط الجانبي. في قسم Build and deployment، اختر المصدر كـ "Deploy from a branch". ثم اختر الفرع (عادةً main) والمجلد (عادةً /(root)). اضغط على Save وانتظر حتى يتم تفعيل Pages.

## الخطوة 4: رفع الملفات

ارفع ملفات موقعك إلى المستودع. تأكد من وجود ملف `index.html` في المجلد المحدد للنشر. يمكنك استخدام Git محليًا أو رفع الملفات مباشرة من خلال واجهة GitHub على الويب.

## الخطوة 5: التحقق من النشر

بعد رفع الملفات، انتقل إلى قسم Actions في المستودع للتحقق من تشغيل سير العمل. إذا سار كل شيء بسلاسة، سظهر الرابط الخاص بموقعك في قسم Pages بعد اكتمال النشر.

## الأسئلة الشائعة

### هل يمكنني استخدام نطاق خاص؟

نعم، يمكنك ربط نطاق خاص بموقع GitHub Pages من خلال إعدادات DNS وإعدادات المستودع. تحتاج إلى إضافة سجلات DNS تشير إلى عناوين GitHub، ثم إدخال اسم النطاق في حقل Custom domain في إعدادات Pages.

### ما هي الملفات المدعومة؟

تدعم GitHub Pages ملفات HTML وCSS وJavaScript وMarkdown والخطوط والصور وملفات PDF والوثائق. لا تدعم الملفات الخادم الديناميكية مثل PHP أو Python أو Ruby.

### كم مساحة المسموح بها؟

الحد الأقصى لحجم الموقع هو 1GB. الحد الأقصى لحجم كل ملف هو 100MB. الحد الأقصى لحجم الـ repository الكلي هو 1GB مع Git LFS.

### هل HTTPS مدعوم؟

نعم، HTTPS مُفعّل تلقائيًا لجميع مواقع GitHub Pages. إذا استخدمت نطاقًا خاصًا، سيتم إصدار شهادة SSL تلقائيًا من خلال Let's Encrypt.
